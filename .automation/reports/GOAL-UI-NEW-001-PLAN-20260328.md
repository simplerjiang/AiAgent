# GOAL-UI-NEW-001 计划书（2026-03-28）

## 1. 目标与背景

本计划用于重构 StockCopilot 的非 Agent UI（明确排除 Agent 工作台/多 Agent 对话功能），解决当前“看起来不够专业、布局拥挤、信息密度失衡、操作路径绕、尺寸不可控”的问题。

核心目标：
- 提升视觉质量：从“功能堆叠页”升级为“可长期使用的交易研究工作台”。
- 提升实用性：关键信息可在更少操作中看到，减少切换和滚动成本。
- 提升可控性：支持位置、宽度、高度、显示密度、面板开合等可配置能力。
- 保持稳定性：不破坏现有后端接口与非 UI 业务行为。

非目标（本次不做）：
- 不改 Agent 工作台相关功能与交互（含多 Agent 协作流、TradingWorkbench、Agent 聊天样式等）。
- 不改量化引擎与 MCP 协议层。
- 不引入自动下单或交易执行能力。

---

## 2. 范围定义

### 2.1 In Scope（纳入重构）
- 顶层壳层与导航：App 顶部品牌区、Tab 导航、页面切换反馈。
- 股票信息页：搜索工具条、顶部市场带、终端主区、图表区、资讯区、交易计划区（仅 UI 结构与样式层面）。
- 情绪轮动页：卡片分区、榜单区、筛选器与信息层级。
- 全量资讯库页：检索、过滤、列表密度、阅读区布局。
- 股票推荐页：推荐列表、上下文摘要、优先级显示。
- 管理设置页：表单可读性、权限提示、状态反馈样式。
- 全局样式系统：色板、字号、间距、圆角、阴影、边框、状态色、响应式断点。
- 交互与可用性：可调宽/高、可折叠、sticky 区、滚动容器边界、空态/错误态统一。

### 2.2 Out of Scope（明确排除）
- Agent 区域/模块交互逻辑与展示重做。
- 后端领域逻辑改造（除支持 UI 必要的小接口字段补充）。
- 数据模型/数据库结构重构。

## 3. 当前痛点（非 Agent）

### 3.1 全局层
- 顶栏与内容层视觉分离不足，产品层级感弱。
- 全局字体、色彩、组件边界风格偏“默认模板感”，不够统一。
- 页面缺乏“主任务优先级”，用户首次进入不清楚该先做什么。

### 3.2 股票信息页
- 信息块过多，用户需要频繁上下滚动与横向扫视。
- 左右区域权重不稳定，图表与关键信息争抢空间。
- 局部可见但不可控：用户难以快速放大自己当前最关心的模块。
- 卡片样式、间距和标题层级不一致，造成视觉噪音。

### 3.3 情绪轮动/资讯库/推荐
- 列表与卡片密度混杂，不同页的扫描节奏不一致。
- 相同语义的状态（刷新中/失败/空数据）表现形式不统一。
- 过滤器区域占用空间偏大，移动端与窄屏体验下降明显。

### 3.4 管理页
- 表单层级与反馈提示偏弱，关键风险信息不够突出。
- 操作成功/失败反馈路径不统一。

---

## 4. 重构原则

- 任务优先：先保证“查股 -> 看图 -> 看市场上下文 -> 看资讯/计划”主链路效率。
- 信息分层：一级（当前决策最相关）> 二级（解释信息）> 三级（补充资料）。
- 可配置布局：支持模块位置、宽度、高度、折叠状态、显示密度持久化。
- 一致性优先：同类型组件遵守统一设计 token 与交互规则。
- 渐进式上线：先做结构与样式底座，再分页面迁移，避免一次性大爆改。
- 回滚安全：每个阶段可独立回退，不阻断主要业务流程。

## 5. 目标 UI 架构（非 Agent）

## 5.1 全局壳层
- 顶部采用“品牌区 + 一级导航 + 全局状态区（连接/刷新/时间）”。
- 内容区采用统一容器宽度策略：
  - `wide`（默认研究模式）
  - `focus`（聚焦阅读模式）
- 导航提供固定高度与滚动保持，减少跨页跳动。

## 5.2 股票信息页（核心页）
- 采用 3 段式结构：
  - 顶部操作带（搜索、快速切换、视图预设）
  - 中部主工作区（图表主舞台 + 关键上下文）
  - 底部扩展区（资讯、计划、历史）
- 支持以下布局控制：
  - 左右分栏拖拽：主舞台/侧栏宽度可调（如 70/30 -> 60/40）。
  - 主舞台高度调节：图表区高度可拖拽，支持最小/最大边界。
  - 卡片折叠/展开：资讯、计划、市场带支持折叠并记忆状态。
  - 单模块聚焦：一键放大某模块（例如图表全宽模式）。

## 5.3 情绪轮动页
- 采用“总览指标条 + 主榜单 + 解释面板”结构。
- 榜单支持密度切换（紧凑/舒适）。
- 指标区支持固定顶部，滚动时保持核心信息不丢失。

## 5.4 全量资讯库页
- 采用“两栏阅读”模式：左侧列表，右侧详情。
- 过滤器折叠到抽屉，默认只露出高频筛选项。
- 长文阅读区支持字号切换和段落间距模式。

## 5.5 推荐页与管理页
- 推荐页强调“推荐理由 + 风险标签 + 市场上下文”并列可读。
- 管理页采用分组表单和 sticky 操作条，减少长表单保存成本。

## 6. 执行阶段与步骤

## Phase 0：基线审计与设计冻结（2-3 天）
- 盘点现状页面信息架构、滚动路径、主要点击热区。
- 输出 UI 问题清单（按 P0/P1/P2 分级）。
- 冻结 design token v1：颜色、字号、间距、圆角、阴影、状态色。
- 冻结布局规范 v1：栅格、断点、可调宽高最小值与最大值。
- 产出：
  - UI 审计清单
  - token 规范文档
  - 非 Agent 范围边界文档

## Phase 1：全局壳层与样式底座（3-4 天）
- 重构全局样式入口：建立统一 CSS 变量与语义化 token。
- 重做 App 顶栏与一级导航视觉层级。
- 统一按钮、输入框、卡片、状态标签样式。
- 建立统一空态/错误态/加载态组件。
- 产出：
  - 全局 UI 基础组件与样式系统
  - 顶层导航与壳层改造完成

## Phase 2：股票信息页结构重构（5-7 天）
- 对股票信息页做布局骨架重排（3 段式）。
- 实现左右分栏拖拽与图表区高度拖拽。
- 支持模块折叠/展开与状态持久化（localStorage）。
- 完成关键模块视觉统一：搜索带、市场带、图表卡、资讯卡、计划卡。
- 产出：
  - 新版股票信息页（非 Agent 区域）
  - 可配置布局能力（位置/宽度/高度）

## Phase 3：情绪轮动 + 资讯库 + 推荐页重构（4-6 天）
- 情绪轮动：重构信息密度与榜单层次。
- 资讯库：两栏阅读与过滤抽屉化。
- 推荐页：理由与风险并列可读、卡片层级优化。
- 产出：
  - 三个核心页面统一交互语言

## Phase 4：管理页与跨页一致性收口（2-3 天）
- 管理页表单分组、反馈统一、风险提示强化。
- 跨页统一验证：边距、标题、按钮、状态、滚动容器行为。
- 产出：
  - 非 Agent 全站视觉一致性通过

## Phase 5：验收与发布准备（2-3 天）
- 单元测试 + Browser MCP 验收 + 桌面打包回归。
- 形成 UI 变更说明与迁移说明（给后续开发者）。
- 产出：
  - 可发布版本
  - 验收报告与风险清单

## 8. 验收标准（DoD）

功能可用性
- 用户可调整关键区域显示位置、宽度、高度，并能持久化。
- 股票信息页主链路（查股、看图、看市场、看资讯/计划）点击路径减少。
- 非 Agent 页面无阻断性布局错乱。

视觉与一致性
- 同类型组件样式一致，状态色语义一致。
- 关键页面在 1366x768、1920x1080、移动窄屏下可用。
- 空态、错误态、加载态在各页统一。

稳定性与质量
- 既有核心单测通过，新增 UI 行为测试通过。
- Browser MCP 验证关键交互无前端 runtime error。
- 桌面打包后可启动并可进入重构页面。

## 10. 风险与应对

- 风险 1：`StockInfoTab.vue` 体量大，重构易引入回归。
  - 应对：以“布局骨架先行 + 逻辑不动”为原则分步替换；每步加定向单测。

- 风险 2：拖拽能力引入复杂状态与跨浏览器差异。
  - 应对：优先使用成熟方案或简化实现；加最小边界和兜底默认值。

- 风险 3：视觉大改导致用户学习成本上升。
  - 应对：保留旧快捷路径入口 1 个版本周期，提供“恢复默认布局”。

- 风险 4：移动端与桌面端冲突。
  - 应对：断点分治，移动端禁用复杂拖拽，改为预设布局切换。
