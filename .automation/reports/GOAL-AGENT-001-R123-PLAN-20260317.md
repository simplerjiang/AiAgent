# GOAL-AGENT-001 R1/R2/R3 Planning Report (2026-03-17)

## EN
### Summary
Split GOAL-AGENT-001 into three executable development slices so the work can be delegated cleanly across parallel developers while keeping a single commander-owned architecture.

### Split Decision
1. R1 is the evidence traceability foundation.
2. R2 is agent role narrowing plus commander hardening.
3. R3 is replay calibration plus acceptance baseline.

### Assignment Model
1. Commander role: architecture owner, contract setter, reviewer.
2. Dev1 primary scope: R1.
3. Dev2 primary scope: R2.
4. R3 starts after the R1/R2 contract stabilizes and is assigned next based on capacity.

### R1 Scope
- URL-first evidence object.
- Backend-first article fetch/clean/summary pipeline.
- `readMode` / `readStatus` semantics.
- Commander evidence trust gate.
- Evidence normalization and history/front-end compatibility.

### R2 Scope
- Re-split sub-agent responsibilities.
- Remove half-commander overlap from stock/sector/financial/trend agents.
- Clean A-share context contamination from `marketReports`.
- Precompute deterministic features in code.
- Add coverage/conflict/degraded-path penalties to commander.

### R3 Scope
- Build replay samples from audit logs and history.
- Align 1/3/5/10-day returns with historical direction/probability/confidence.
- Compute hit rate, Brier score, grouped win rate, traceability rate, repair rate, contamination rate, and revision-explanation completeness.
- Turn these metrics into a reusable acceptance baseline.

### Directive Cleanup
- Rewrote `.automation/chatgpt_directives.md` so it no longer centers on historical Step 4.x and chart branches.
- The directives file now focuses only on GOAL-AGENT-001-R1/R2/R3, the commander/developer collaboration model, and the still-active architecture constraints.

### Planned Validation

2. Check diagnostics on the changed markdown files.
