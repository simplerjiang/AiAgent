# GOAL-013 Step 2.5 Planning Report (2026-03-13)

## EN
### Summary
Planned Step 2.5 to resolve three production issues after the Step 2.4 rollout: market-news UX should move back into the page flow with an expanded reading mode, stale overseas RSS items must be filtered aggressively, and AI enrichment retries must continue even when local facts are already considered fresh.

### Planned Scope
- Replace the current sticky/floating market-news banner with an in-flow market-news panel and add an expand/fullscreen reading affordance so dense translated headlines remain readable without relying on the AI sidebar.
- Add upstream cache busting for overseas RSS fetches via a timestamp query suffix when requesting feeds.
- Enforce hard freshness filtering for market RSS items and domestic market facts so any item older than the accepted recency window is dropped before persistence.
- Keep market/news ordering strictly descending by publish time after parsing and before dedupe/truncation.
- Fix the refresh flow so pending `IsAiProcessed == false` rows for `market`, `stock`, and `sector` are still sent through the batch AI enrichment path even when data freshness checks would otherwise return early.
- Extend tests for RSS recency filtering, refresh retry behavior, and the updated market-news UI interaction.

### Planned Validation
1. `dotnet test backend/SimplerJiangAiAgent.Api.Tests/SimplerJiangAiAgent.Api.Tests.csproj --filter "LocalFactIngestionServiceTests|RssMarketNewsParserTests|LocalFactAiEnrichmentServiceTests|QueryLocalFactDatabaseToolTests"`
2. `npm run test:unit -- src/modules/stocks/StockInfoTab.spec.js`
3. `npm run build`
4. `node frontend/scripts/edge-check-goal013.mjs`

### Risks Noted
- The current `EnsureMarketFreshAsync` / `EnsureFreshAsync` early-return logic can suppress retry of pending AI enrichment rows; this needs to be fixed without causing duplicate expensive requests.
- UI changes must preserve the focus-mode layout and not reintroduce chart height regression from GOAL-012.
- RSS cache busting must remain deterministic and not break feed URLs that already contain query strings.

## ZH
### 摘要
规划执行 Step 2.5，解决 Step 2.4 上线后暴露的三个实机问题：大盘资讯需要回到页面流内并支持放大阅读，海外 RSS 旧闻要被强力过滤，本地事实的 AI 清洗重试不能再被“已有新鲜数据”这类早退逻辑绕过。

### 规划范围
- 把当前 `sticky/floating` 的大盘资讯条带改成页面流内嵌卡片，并增加展开/放大阅读入口，便于集中查看翻译后的密集宏观资讯，而不依赖 AI 侧栏。
- 海外 RSS 请求增加时间戳 query 参数做去缓存处理，减少上游静态缓存污染。
- 为海外 RSS 与国内大盘事实增加硬时效过滤；超出允许时效窗口的新闻在入库前直接丢弃。
- 解析后、去重前统一按发布时间倒序排序，确保保留的始终是最新资讯。
- 修复 `EnsureMarketFreshAsync` / `EnsureFreshAsync` 的早退逻辑：即使数据已算“新鲜”，只要存在 `IsAiProcessed == false` 的 `market`、`stock`、`sector` 行，也必须继续走批量 AI 清洗补标。
- 补充 RSS 时效过滤、AI 重试补漏和新版大盘资讯交互的测试。

### 风险记录
- 当前 `EnsureMarketFreshAsync` / `EnsureFreshAsync` 的早退会让待补标的 AI 新闻永远得不到重试，修复时要避免把昂贵请求重复打满。
- 前端布局调整必须保住 GOAL-012 已修好的专注模式和图表高度，不得重新引入撑高或遮挡问题。
- RSS 去缓存参数需要兼容原本已带 query 的链接，不能把上游 URL 拼坏。
