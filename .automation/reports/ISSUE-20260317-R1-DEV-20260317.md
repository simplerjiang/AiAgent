# ISSUE-20260317-R1 Development Report

## English

### Scope
- Fix wrong preset take-profit / target draft values when the saved commander chart prices are no longer directionally valid against the current price.
- Restore missing probability and scoring rows in the multi-agent table.
- Reduce empty sector-news panels by providing a stable local fallback when symbol-level sector news rows are missing.
- Pass richer local facts into agent context so financial and news agents do not lose AI-enriched metadata and cached fundamentals.

### Actions
- Updated `TradingPlanDraftService` to validate directional target / take-profit candidates against commander `metrics.price` when available.
- Extended `StockAgentResultNormalizer` to flatten probability and scoring aliases into a stable `metrics` shape.
- Extended local-fact models and projection so agent context now keeps `Sentiment`, `AiTarget`, `AiTags`, and cached `FundamentalFacts`.
- Updated `QueryLocalFactDatabaseTool` to:
  - include cached company fundamentals from `StockCompanyProfiles`
  - synthesize sector fallback items from `SectorRotationSnapshots` and `SectorRotationLeaderSnapshots` when symbol-level `LocalSectorReports` are empty
- Refined `QueryLocalFactDatabaseTool` fallback again so it searches the most recent matching sector snapshot across snapshot batches and can fall back via leader-symbol membership when sector names do not line up with the company profile.
- Fixed a remaining sidebar-sector-context gap for symbols like `sz000021`: the fallback now preserves snapshot recency ordering correctly and, when no sector snapshot or leader membership can be resolved, falls back to recent market reports as a non-empty sector-context bucket.
- Updated `StockAgentPanels.vue` and `agentFormat.js` so older nested result shapes still render the expected metric rows.
- Extended `StockAgentPanels.vue` compatibility again so saved commander history records still render `entryScore`, `valuationScore`, `positionPercent`, and target / stop / take-profit values from `recommendation`.
- Added regression tests for all new behaviors.

## 中文

### 本次范围
- 修正“基于此分析起草交易计划”里止盈价 / 目标价在当前价面前明显失真的问题。
- 恢复多 Agent 面板里缺失的上涨/下跌概率、入场评分、估值评分等指标行。
- 解决 symbol 维度 `LocalSectorReports` 缺失时板块资讯经常空白的问题。
- 把本地事实库中的 AI 富信息和缓存基本面继续传给 agent，避免在投喂链路中丢失。

### 已完成动作
- 修改 `TradingPlanDraftService`：当 commander 结果里带有当前价时，只接受方向上合理的目标价/止盈价候选。
- 修改 `StockAgentResultNormalizer`：把概率、评分、仓位、止盈/目标/止损等别名统一折叠到稳定的 `metrics`。
- 修改本地事实模型与 projection：agent 上下文不再丢弃 `Sentiment`、`AiTarget`、`AiTags`，并追加缓存 `FundamentalFacts`。
- 修改 `QueryLocalFactDatabaseTool`：
  - 读取 `StockCompanyProfiles` 的缓存基本面事实
  - 当 symbol 级板块资讯为空时，用 `SectorRotationSnapshots` 和 `SectorRotationLeaderSnapshots` 合成本地板块快照兜底
- 继续收紧 `QueryLocalFactDatabaseTool` 的兜底逻辑：不再只看“全库最新一批”快照，而是跨快照批次查找最近匹配的板块；当行业名对不上时，再通过 leader symbol 反查最近关联板块。
- 修复 `QueryLocalFactDatabaseTool` 里板块兜底排序被后续 `OrderBy` 覆盖的问题；并为 `sz000021` 这类既没有板块报告、也没有可命中轮动快照的样本追加“市场环境兜底”，保证侧栏 `板块上下文` 不再整块空白。
- 修改 `StockAgentPanels.vue` 与 `agentFormat.js`：兼容旧历史里的嵌套 probability shape，保证面板继续显示关键指标。
- 继续补强 `StockAgentPanels.vue`：旧 commander 历史若把评分/仓位/价位放在 `recommendation` 里，也会正确渲染 `入场评分 / 估值评分 / 建议仓位 / 目标价 / 止盈价 / 止损价`。
- 补齐对应回归测试。
