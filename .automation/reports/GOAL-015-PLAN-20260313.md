# GOAL-015 Step 3 Planning Report (2026-03-13)

## EN
### Summary
Planned Step 3 to enrich quote/fundamental facts, persist slower company-profile data, inject the new fields into the multi-agent context, and replace the old recommendation-centric commander schema with an opinion-centric decision-assist schema.

### Planned Scope
- Extend stock quote/domain models with `floatMarketCap`, `peRatio`, `volumeRatio`, `shareholderCount`, and `sectorName`.
- Upgrade the Eastmoney realtime quote request to include `f10`, `f117`, and `f162` class fields needed for volume ratio, float market cap, and dynamic PE parsing.
- Expand the existing Eastmoney company-profile parsing path so F10 data can provide sector and shareholder-count data, then cache those slower-changing fields in a dedicated persistence model.
- Persist the new fields into cacheable backend data so `/api/stocks/detail`, `/api/stocks/detail/cache`, sync jobs, and agent context all read a consistent shape.
- Redesign the commander prompt/output contract from `recommendation.action/...` into the new JSON contract with `analysis_opinion`, `confidence_score`, `trigger_conditions`, `invalid_conditions`, and `risk_warning`.
- Inject the new quote fundamentals into commander and sub-agent prompt context as structured JSON instead of relying on terse price-only context.
- Update the Vue stock information summary strip and AI result panel to render the five new fundamentals and the new commander response structure cleanly.
- Add/update backend and frontend tests to cover parser changes, schema normalization, and panel rendering.

### Planned Validation
1. `dotnet test backend/SimplerJiangAiAgent.Api.Tests/SimplerJiangAiAgent.Api.Tests.csproj --filter "FullyQualifiedName~EastmoneyStockParserTests|FullyQualifiedName~EastmoneyCompanyProfileParserTests|FullyQualifiedName~StockAgentPromptBuilderTests|FullyQualifiedName~StockAgentResultNormalizerTests|FullyQualifiedName~StockSyncServiceTests"`
2. `npm --prefix frontend run test:unit -- src/modules/stocks/StockInfoTab.spec.js src/modules/stocks/StockAgentPanels.spec.js`
3. `npm --prefix frontend run build`

### Risks Noted
- The repo currently stores cached quotes in `StockQuoteSnapshots`; schema expansion must stay compatible with the existing cache detail endpoint and sync jobs.
- Commander history / guardrail logic still reads old `recommendation` fields, so the schema migration must preserve deterministic direction extraction.
- Frontend chart overlays currently derive support/resistance from old commander fields; if the new schema removes them completely, overlays must fall back deterministically.

## ZH
### 摘要
规划执行 Step 3，核心是三件事：补齐股票深度盘面与基本面字段、把这些事实稳定写入后端缓存与 Agent 上下文、并把原来偏“机械操作指令”的 commander 输出改造成“分析意见 + 条件 + 风险限制”的投顾辅助结构。

### 规划范围
- 扩充股票行情/领域模型，增加 `floatMarketCap`、`peRatio`、`volumeRatio`、`shareholderCount`、`sectorName` 五个核心字段。
- 升级东方财富实时行情请求字段，补齐量比、流通市值、动态 PE 所需的字段解析。
- 扩展现有东方财富公司资料/F10 解析链路，提取所属板块与股东人数，并为这些慢变字段建立专门缓存持久化模型。
- 将新增字段统一接入后端缓存链路，确保 `/api/stocks/detail`、`/api/stocks/detail/cache`、同步任务与多 Agent 上下文使用同一份结构。
- 将 commander Prompt/输出契约从旧的 `recommendation.action/...` 重构为新的 `analysis_opinion`、`confidence_score`、`trigger_conditions`、`invalid_conditions`、`risk_warning` 结构。
- 把新增的 PE、量比、流通市值、板块等数据以 JSON 结构注入 commander 与子 Agent 的上下文，而不是继续只给简单价格信息。
- 更新 Vue 股票信息页概要栏与 AI 结果面板，完整展示这 5 个基本面字段和新的 commander 输出结构。
- 增补后端与前端测试，覆盖解析器、schema 归一化和面板渲染。

### 风险记录
- 当前仓库把缓存行情落在 `StockQuoteSnapshots`，扩字段时必须兼容既有 cache detail 接口与同步任务。
- commander 历史与一致性守护仍依赖旧 `recommendation` 字段，改 schema 时需要保留可确定的方向提取逻辑。
- 前端图表叠加线仍从旧 commander 字段推导支撑/压力；如果新 schema 完全移除这些字段，必须提供确定性回退，避免图表功能回退。
