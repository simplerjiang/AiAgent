# GOAL-016 DEV REPORT - PACKAGED SINGLE-MACHINE FOUNDATION - 2026-03-20

## English
- Scope: land the first runnable foundation for a packaged Windows single-machine build that no longer requires SQL Server installation, while keeping LLM secrets user-configured only.
- Backend actions:
  - Added `AppRuntimePaths` to centralize writable runtime paths for SQLite, logs, bundled frontend assets, and local LLM settings.
  - Switched base runtime database provider handling to support `Sqlite` and default to a local SQLite file when no connection string is supplied.
  - Moved runtime log and LLM settings writes from the install directory into `%LOCALAPPDATA%\SimplerJiangAiAgent`.
  - Guarded SQL Server-specific schema initializer logic so SQLite startup does not execute T-SQL repair scripts.
  - Fixed `StockCompanyProfile.FundamentalFactsJson` model configuration so SQL Server-only `nvarchar(max)` is not emitted for SQLite schema creation.
- Desktop actions:
  - Updated the WinForms/WebView2 host to detect an existing healthy backend, or start the packaged backend from `Backend/` automatically.
  - Added production environment injection, local data-root injection, health polling, and owned backend-process cleanup on app close.
- Packaging actions:
  - Added `scripts/publish-windows-package.ps1` to build frontend assets, publish backend and desktop binaries, copy frontend `dist`, and seed only non-secret LLM defaults.
  - Documented the local writable data root and packaged LLM-settings behavior in `README.md`.
- Test commands:
  - `dotnet test .\backend\SimplerJiangAiAgent.Api.Tests\SimplerJiangAiAgent.Api.Tests.csproj`
  - `dotnet build .\desktop\SimplerJiangAiAgent.Desktop\SimplerJiangAiAgent.Desktop.csproj`
  - `& .\scripts\publish-windows-package.ps1 -SelfContained:$false`
  - Published-backend smoke validation on `http://localhost:5128/api/health`
  - `Test-Path .\artifacts\windows-package\Backend\App_Data\llm-settings.local.json`
- Results:
  - Backend tests passed: `174/174`.
  - Desktop build passed.
  - Packaging script succeeded and produced `artifacts\windows-package`.
  - Published backend returned `{"status":"ok"}` from `/api/health` and created the SQLite database file under the configured data root.
  - Package verification confirmed `llm-settings.local.json` was not included.
- Issues:
  - The first published-backend smoke run failed because EF still emitted `nvarchar(max)` for SQLite on `StockCompanyProfiles.FundamentalFactsJson`. This was fixed in the same task by making the column type conditional on SQL Server only.

## 中文
- 范围：落下第一阶段“可打包、可运行、无需安装 SQL Server”的 Windows 单机版基础，同时继续保持 LLM 密钥只由用户自行配置，不随安装包分发。
- 后端动作：
  - 新增 `AppRuntimePaths`，统一管理 SQLite 文件、日志目录、打包前端目录和本地 LLM 配置路径。
  - 扩展数据库 provider 处理，支持 `Sqlite`，并在未显式提供连接串时默认落到本地 SQLite 文件。
  - 将运行期日志和 LLM 设置写入位置从安装目录迁移到 `%LOCALAPPDATA%\SimplerJiangAiAgent`。
  - 将 SQL Server 专属的启动期 schema 修复逻辑改为 provider-aware，避免 SQLite 启动时执行 T-SQL。
  - 修复 `StockCompanyProfile.FundamentalFactsJson` 的 EF 建模，让 `nvarchar(max)` 只在 SQL Server 下生效，确保 SQLite 能成功建表。
- 桌面端动作：
  - 更新 WinForms/WebView2 桌面宿主：优先复用现有健康后端，否则自动从 `Backend/` 目录拉起打包后的后端。
  - 补齐 `Production` 环境变量注入、本地数据目录注入、健康检查轮询，以及关闭应用时清理自启动后端进程。
- 打包动作：
  - 新增 `scripts/publish-windows-package.ps1`，串起前端构建、后端发布、桌面发布、`dist` 复制，以及仅投递非敏感 `llm-settings.json`。
  - 在 `README.md` 中补充本地数据目录和发布版 LLM 设置行为说明。
- 测试命令：
  - `dotnet test .\backend\SimplerJiangAiAgent.Api.Tests\SimplerJiangAiAgent.Api.Tests.csproj`
  - `dotnet build .\desktop\SimplerJiangAiAgent.Desktop\SimplerJiangAiAgent.Desktop.csproj`
  - `& .\scripts\publish-windows-package.ps1 -SelfContained:$false`
  - 对发布版后端执行 `http://localhost:5128/api/health` 烟测
  - `Test-Path .\artifacts\windows-package\Backend\App_Data\llm-settings.local.json`
- 结果：
  - 后端测试通过：`174/174`。
  - 桌面项目构建通过。
  - 打包脚本执行成功，产出 `artifacts\windows-package`。
  - 发布版后端 `/api/health` 返回 `{"status":"ok"}`，并在配置的数据目录下自动创建 SQLite 数据库文件。
  - 包内容检查确认没有带上 `llm-settings.local.json`。
- 问题：
  - 首次发布版后端烟测失败，根因是 EF 在 SQLite 建表时仍输出了 `StockCompanyProfiles.FundamentalFactsJson` 的 `nvarchar(max)`。该问题已在同一任务内修复，改为只在 SQL Server 下设置该列类型。
