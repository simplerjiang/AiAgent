# GOAL-013 Development Report (EN + ZH)

## EN
### Summary
Completed Step 2 for GOAL-013: added a local-first fact hub for A-share stock announcements/news, sector/market context, controlled internet routing for stock agents, and precise stock-page news filtering.

### Implemented Changes
- Added `LocalStockNews` and `LocalSectorReport` entities, registered them in `AppDbContext`, and generated migration `20260312090154_AddLocalFactTables`.
- Implemented `LocalFactIngestionService` + `LocalFactIngestionWorker` under Infrastructure:
  - Eastmoney announcement JSON API for stock notices
  - Eastmoney company survey API for sector/industry metadata
  - existing `SinaCompanyNewsParser` for company news
  - Sina roll feed filtering for sector/market context
- Added `QueryLocalFactDatabaseTool` and routed stock agents to local facts for domestic A-share single-symbol scenarios.
- Added code-level stock-agent internet routing policy to keep external search available only for eligible future non-domestic scenarios.
- Added `GET /api/news?symbol={code}&level=stock|sector|market`.
- Updated `StockInfoTab.vue` to render local stock/sector/market news buckets tied to the active symbol.
- Added backend tests for parsers/query tool/routing policy and frontend tests for local news bucket requests.

### Database Notes
- `dotnet ef migrations add AddLocalFactTables` generated the migration successfully.
- The first `dotnet ef database update` failed because the generated migration accidentally included legacy governance tables already created by schema initializers.
- Fixed by trimming `AddLocalFactTables` to GOAL-013 tables only, then re-running `dotnet ef database update` successfully.
- Added `LocalFactSchemaInitializer` so existing local databases running under the repository's `EnsureCreated + initializer` startup path still get the new tables.
