# GOAL-015 Step 3 Development Report (2026-03-13)

## EN
### Scope
- Extended the stock quote/cache model with float market cap, PE ratio, volume ratio, shareholder count, and sector name.
- Added a dedicated company-profile persistence table for slower-changing Eastmoney F10 facts.
- Reworked the commander contract from `recommendation.*` to the new opinion-centric schema and updated the Vue stock terminal to render it.

### Development
- Backend market-data expansion:
  - `StockQuoteDto` now carries `floatMarketCap`, `volumeRatio`, `shareholderCount`, and `sectorName` in addition to existing quote facts.
  - `StockQuoteSnapshot` now persists `PeRatio`, `FloatMarketCap`, `VolumeRatio`, `ShareholderCount`, and `SectorName`.
  - Added `StockCompanyProfile` plus `StockMarketDataSchemaInitializer` so existing databases get the new table/columns even though startup still uses `EnsureCreated()`.
- Eastmoney integration:
  - Upgraded realtime quote parsing to request and parse `f117`, `f162`, and `f10` with tolerant handling for missing `f10`.
  - Extended company-profile fetching to combine `CompanySurveyAjax` with `ShareholderResearch/PageAjax`, extracting sector and shareholder count.
  - Updated `CompositeStockCrawler` to merge fields across multiple sources so Eastmoney-only fundamentals are not dropped behind Tencent/Sina/Baidu source order.
- Persistence and cache path:
  - `StockSyncService` now upserts both quote snapshots and company-profile rows.
  - `/api/stocks/detail/cache` now reconstructs the expanded quote shape from snapshot + company-profile cache.
- Multi-agent schema refactor:
  - Replaced commander prompt/repair schema from `recommendation.action/...` to `analysis_opinion`, `confidence_score`, `trigger_conditions`, `invalid_conditions`, and `risk_warning`.
  - Updated commander history extraction and consistency guardrails to infer direction from the new textual opinion schema.
  - Added compatibility fallbacks so normalization can still tolerate partially old-shaped commander outputs during transition.
- Frontend:
  - `StockInfoTab.vue` now shows the five Step 3 fundamentals in the terminal summary.
  - `StockAgentPanels.vue` now renders commander output as narrative conclusion/trigger/invalidation/risk blocks instead of the old operation-plan table.
  - AI support/resistance overlay fallback now parses tagged prices from textual commander conditions and then falls back to trend forecast extremes.

### 开发
- 后端行情数据扩展：
  - `StockQuoteDto` 现已承载 `floatMarketCap`、`volumeRatio`、`shareholderCount`、`sectorName` 等新增事实。
  - `StockQuoteSnapshot` 已新增 `PeRatio`、`FloatMarketCap`、`VolumeRatio`、`ShareholderCount`、`SectorName` 持久化字段。
  - 新增 `StockCompanyProfile` 与 `StockMarketDataSchemaInitializer`，保证在当前仍使用 `EnsureCreated()` 的启动路径下，旧数据库也能自动补齐 Step 3 表结构。
- 东财接入升级：
  - 实时行情已升级解析 `f117`、`f162`、`f10`，并对 `f10` 缺失场景做了容错。
  - 公司资料抓取已组合 `CompanySurveyAjax` 与 `ShareholderResearch/PageAjax`，提取所属板块与股东户数。
  - `CompositeStockCrawler` 不再只吃第一个成功源，而是按字段合并多源结果，避免东财独有基本面被腾讯/新浪/百度前置顺序覆盖掉。
- 持久化与缓存链路：
  - `StockSyncService` 已同步写入扩展行情快照和公司资料表。
  - `/api/stocks/detail/cache` 现在会从快照表 + 公司资料表还原完整的 Step 3 行情结构。
- 多 Agent schema 重构：
  - commander Prompt 与 repair schema 已从 `recommendation.action/...` 切换为 `analysis_opinion`、`confidence_score`、`trigger_conditions`、`invalid_conditions`、`risk_warning`。
  - commander 历史解析与一致性守护逻辑已改为基于文本化结论推断方向，不再依赖旧 recommendation 字段。
  - 同时加入兼容性兜底，过渡期若模型偶尔吐出半旧结构，也会被 normalizer 补齐到新结构。
- 前端：
  - `StockInfoTab.vue` 已在终端概要区展示 5 个 Step 3 基本面字段。
  - `StockAgentPanels.vue` 已改为展示 commander 的结论、触发、失效和风险文本块，不再渲染旧“操作计划”表。
  - 图表 AI 支撑/压力线已改为从 commander 文本条件里提取带语义的价格，再回退到 trend forecast 极值。
