# ISSUE-20260310-P1 Plan - News Anti-Pollution Guardrails

## Goal
减少中文互联网低质量信息对 Agent 决策的污染风险，确保结论可追溯、可降级、可审计。

## Work Breakdown
1. 定义来源分层策略
   - authoritative: 交易所公告、监管网站、上市公司 IR
   - preferred: 主流财经媒体（需时间戳）
   - fallback: 行业聚合与二级转载（需降权）
   - blocked: 自媒体谣言/无来源内容（默认禁用）
2. 定义时效策略
   - 默认证据窗口 72h
   - 不足证据时允许扩窗到 7d（必须在输出标注）
3. 定义结论降级策略
   - 任一关键证据缺来源或发布时间 -> neutral/insufficient-data
4. 落地到提示词与后端聚合逻辑
5. 增加单测与回归测试

## Data Contract Additions
- evidenceQualitySummary
- sourceTierBreakdown
- recencyWindowUsed
- downgradeReason

## DoD
- 方向性建议必须基于可追溯证据
- 低质量证据不会触发高置信结论
- 单测覆盖 3 类边界场景并通过

## Implementation Blueprint (Code-Level)
1. 决策入口先拦截（先风控，后增强）
   - 在新闻上下文组装入口先做来源分层、时间窗过滤、证据完整性检查。
   - 当关键证据不满足“source + publishedAt”时，不允许方向性高置信输出。
2. 聚合层统一降级
   - 在多 Agent 汇总阶段统一执行降级规则，避免子 Agent 结果绕过策略。
   - 降级后输出 action=observe、rating=neutral，并写入 downgradeReason。
3. Prompt 与结果结构同时收口
   - Prompt 明确禁止基于 blocked/无时间戳证据给出强结论。
   - 结果 JSON 强制包含证据质量字段，保证前端和审计链路可见。

## Boundary Test Cases
1. blocked-only
   - 输入均为 blocked 源，预期：`neutral/observe` + `downgradeReason=blocked_only_sources`
2. untimestamped-only
   - 关键证据无发布时间，预期：`neutral/observe` + `downgradeReason=missing_timestamp`
3. mixed-source
   - authoritative/preferred/fallback 混合，预期：按可信证据主导；不足时触发 7d 扩窗并标注
