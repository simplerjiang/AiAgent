# ISSUE-20260310-P2 Plan - Internal News Knowledge Base + Scheduler

## Goal
构建可复用新闻资产（市场/板块/个股），支持定时采集、去重落库、供 Agent 检索。

## Dual-Track Information Model
1. 个股事实信息轨（Stock Fact Track）
   - 内容范围：公司大事、公告、核心题材、分红融资、财务摘要、管理层变动。
   - 特点：结构化程度高、可程序化稳定获取、适合定时入库和长期画像。
   - 用途：作为多 Agent 分析中的“硬证据底座”。
2. 板块情报信息轨（Sector Intelligence Track）
   - 内容范围：板块政策、产业链事件、资金风格、景气变化、龙头异动扩散。
   - 特点：异构且变化快，来源稳定性弱于个股公告，需要动态治理。
   - 用途：作为交易节奏与轮动判断的“情境证据”。

## Dependency
- 本任务依赖 `ISSUE-20260310-P0` 来源治理基座：
   - 仅消费通过治理的来源（authoritative/preferred/fallback）
   - 来源异常时自动降级/隔离，采集链路自动切换

## Work Breakdown
1. 数据库设计
   - news_items: 主体内容、来源、时间戳、等级、哈希去重键
   - news_entities: 关联证券/板块/大盘标签
   - news_tags: 主题标签与事件分类
   - news_runs: 采集任务运行记录（开始/结束/状态/错误）
   - stock_facts: 个股事实快照（财务、分红融资、重大事项、题材标签）
   - sector_signals: 板块信号快照（强度、扩散度、资金偏好、催化事件）
2. 采集器实现
   - 多源抓取、字段标准化、时间解析、正文清洗
   - 采集源从 `source_registry` 动态加载，禁止写死来源列表
   - 个股轨与板块轨分离采集、分离评分、分离回退策略
3. 去重与质量门禁
   - URL + 标题归一化哈希
   - 无时间戳或来源不明标记为低可信
   - 当 `source_registry.status=quarantine` 时自动停用该源
4. 定时任务实现
   - 间隔调度 + 重试退避 + 最大重试上限
   - 增加日任务：LLM 候选源发现、程序化候选验证、来源重评分与自动升降级
5. 查询接口与 Agent 接入
   - 按 symbol/sector/index + 时间窗检索
   - 默认返回来源等级与验证状态，供 P1 分析策略使用
   - 多 Agent 上下文拼装时，默认同时注入：
     - stock facts（过去 30/90/365 天可配置窗口）
     - sector intelligence（过去 72h 为主，证据不足扩窗 7d）

## Sector Source Strategy (Practical)
1. 优先来源（authoritative/preferred）
   - 交易所与监管公告、行业协会/主管部门公告、上市公司公告汇总。
   - 主流财经媒体的行业频道（必须带可解析发布时间和原始链接）。
2. 可选来源（fallback）
   - 行业垂直媒体、二级聚合资讯（仅作补充，不可单独驱动高置信结论）。
3. 不稳定来源处理
   - 通过 P0 `source_registry` 做动态评分与自动隔离，避免硬编码依赖。

## If Good Sector Sources Are Limited
- 用“本地可得数据”反推板块信号，降低对外部新闻单点依赖：
  - 同板块成分股涨跌分布、涨停/跌停占比、成交额放量比、龙头-跟风扩散速度。
  - 将上述量化信号与少量高可信行业新闻融合，形成 sector_signals。
- 当外部板块新闻覆盖不足时，系统自动将结论降级为 neutral，并标注证据不足。
